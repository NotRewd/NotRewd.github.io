Pixel Shifter
Code Documentation

All the documentation needed for creating fully functional maps .

Definitions

[
     {
          "time" :  1 ,
          "type" : "enemy",
          "name" : "striker"
     }
]

Definitions are keywords that are used to specify the event that you want to happen during the game. For example: "time" is a definition and 1 it's the definition's value. Definitions are always in quotes followed by a colon.

Time & Beat

[
     {
          "time" :  1 ,
          "type" : "enemy",
          "name" : "striker"
     }
]
                 OR
[
     {
          "beat" :  2.13 ,
          "type" : "enemy",
          "name" : "striker"
     }
]

Time and beat are definitions that are used to determine when will the event take place. Time is in seconds and it counts upwards from the moment the level started. Beat, on the other hand, relies on the beat of the song. You can use only one of those in a single event. It is also not recommended to use them both in a single map.

Types

[
     {
          "time" :  1 ,
          "type" : "enemy",
          "name" : "striker"
     }
]
                 OR
[
     {
          "time" :  1 ,
          "type" : "effect",
          "name" : "camerashake"
     }
]

Currently, there are two types: "enemy" and "effect". Type "enemy" is an object that tries to mainly kill the player; hovever, the "effect" type focuses more on the visual side rather than the obstacle side.

Enemies

Enemies belong to the "name" definition.

Striker

Striker is just a regular enemy with no special abilities.

Dasher

Dasher can use his ability to dash towards the player.

Ant

Ant is very small and spawns close to the player.

Giant

Giant keeps changing to different sizes.

Splitter

Splitter can split into multiple smaller parts.

Beam

Beam instantly tracks down the player.

Camera Shake Effect

[
     {
          "time" :  1 ,
          "type" : "effect",
          "name" : "camerashake" ,
           "shakeduration" :  2 , (Optional)
           "shakeamount" :  0.1, (Optional)
           "decreasefactor" :  1 (Optional)
     }
]

This effect shakes the screen. The  "shakeduration" definition is the duration of the shake in seconds. You can also change the intensity in  "shakeamount" definition. The  "decreasefactor"  defines the decreasing factor of the camera shake's duration. You do not need to include optional definitions if you want the default camera shake.

Bars Effect

[
     {
          "time" :  1 ,
          "type" : "effect",
          "name" : "bars" ,
           "barstype" :  "horizontal",   (Optional)
           "barsduration" :  3 (Optional)
     }
]

Bars effect creates horizontal or vertical bars on the screen. If a player touches them, they will die. There are three types of bars: "horizontal", "vertical" or "both".
